"Code is like humor. If you have to explain it, it's bad."

Answer "why", not "what".
- Decisions that are not immediately obvious
- Tricky logic that needs clarification
- Info not gleaned from code itself

Comments can be a sign of bad, convoluted tangled logic. 
You might need to structure it in a more obvious way.

Use tooltips for inspector comments
`[Tooltip("The amount of side-to-side friction.")]`
`public float Grip;`

Use summary XML tags in front of public methods and functions.
