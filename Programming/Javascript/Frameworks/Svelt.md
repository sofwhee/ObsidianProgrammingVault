## Review

**TLDR: I wouldn't use Svelt cause I don't like the syntax.**

It was really important for me because it was the first framework that challenged my die-hard "React bro" nature.

"Rethinking Reactivity" - phenomenal talk.

It showed me that React wasn't really reactive.

It showed me all the things React *didn't do.* Things which made my dev experience a bit more painful!
- React had no opinon on styles
- React had no opinion on bundlers
	- Now they throw webpack in kinda haphazardly

Single file components problems:
I want the ability to not lose the context of the file I'm in and break it out quickly.

When you use Svelt you're stuck with single file components... which makes you do crazy things with your markup syntax.

It reinvents JS inside HTML... which some dislike. LIke me.

I don't wanna learn a different JavaScript.
