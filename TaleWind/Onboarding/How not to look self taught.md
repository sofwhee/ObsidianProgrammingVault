#### Plan before you code
- Layout what you're building
- Layout what the requirements are
- Think about how it fits into the existing codebase
	- what context you can use
	- what existing code you can use
	- what you can do to avoid reinventing
	- Readability
	- Maintainability
	- Scalability
#### Don't make huge changes at once. 
- Don't make massive pull requests
	- 1 cohesive feature you're working on - not multiple
	- If it's a big feature, break it into parts
	- if you sketch or plan it out first, you probably ended up with a steplist
	- every step you make, test it, and commit it
- Don't ever push huge code that could cause a crash or break the build
- Push up your branch, make a pull request to be reviewed, and then someone from master reviews it
- Commit at each check mark
#### Don't learn new languages you don't need all the time
- build in psuedocode
- then in new language
#### Don't be embarassed to say "i'm in the middle of something, I'll get back to you once I'm free from completing this task. I want to do this right!"
#### Don't avoid critiques.
