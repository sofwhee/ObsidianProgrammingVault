Functional testing
- Unit : (Functions or methods) in isolation
- Integration : (Modules or services) working together well when combined
- System : (Complete, integrated software as a whole)
- Acceptance (UAT) : Does it satisfy business needs
- Regression : Check if anything was broken after changes

Non-functional testing
Beyond basic function. User experience, stability.
- Performance : speed, stability under workload
	- Load : behaviour under expected user volume
	- Stress : Finds breaking point
- Security : Identify vulnerabilities to protect data from unauthorized access
- Usability : User-friendliness. Intuitiveness.
- Accessibility : disabilities

Testing techniques
- White-Box : With full knowledge. Usually by developers.
- Black-box : End-user. Purely inputs and outputs.
- Grey-Box : Hybrid, partial knowledge. Smarter tests.
- Smoke : Most critical functions.
- Exploratory : Unscripted. Manual. Edge-case bugs.
