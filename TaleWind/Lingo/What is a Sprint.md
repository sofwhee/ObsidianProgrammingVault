A sprint is a time-boxed period of work by an Agile team.

Goal of having sprints:
- Teams can pivot quickly to changing requirements
- Faster delivery: Frequent incremental releases, preventing long dev cycles
- Continuous feedback
- Regular testing and review cycles to catch issues early

User stories - Descriptions of features from a user's perspective.
Sprint Backlog - list of User Stories committed to the sprint

Test case - Detailed steps for testing
Test Scenario - High level description of test case
Unit Test - code for testing an individual component

Sprint planning
- Team defines goal
- Team selects backlog tasks to complete
Daily Sprint Standup (Scrum) 
- Short meeting to discuss progress, things blocking progress.
Sprint Review
- Demonstrate working software to stakeholders for feedback.
Sprint Retrospective
- Team identifies improvements for next sprint
Estimation meeting
- This is where we give estimations per ticket
- This is where we change priority of tickets
