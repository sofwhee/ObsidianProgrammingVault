- Cluster servers - Share workloads to improve reliability by preventing downtime through a method called "automatic failover"
	- Automatic failover - if one server fails, another takes over
	- Useful for heavy traffic, databased, critical apps
	- If one server crashes, the cluster continues to function
	- Scalable: CPU, memory, storage can be increased by adding more nodes
	- Faster processing times thanks to workload distribution
	- High Availability Clusters (HA) - focused on uptime. Failover.
	- Load-Balancing Clusters - distributes incoming traffic/workloads across all nodes to prevent any single server from being overwhelmed
	- High-Performance Computing (HPC) Clusters - Combine processing power of many nodes to solve complex computational problems
